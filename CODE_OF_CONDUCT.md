# 社区行为准则 Code of Conduct

## 我们的承诺

为了让每个人都能享受一个开放、友好的环境，无论年龄、体型、残疾、族裔、性别特征、性别认同和表达、经验水平、教育程度、社会经济地位、国籍、个人外貌、种族、宗教信仰、性取向如何，**代码考古学 CodeArchaeology** 项目的贡献者和维护者承诺：让参与本项目和社区的每一个人都不受骚扰。

不论项目形式——线上或线下会议、Issue、PR、Discussion、社交媒体——我们都致力于让社区成为**对所有人**都安全的体验。

本准则改编自 [Contributor Covenant 2.1](https://www.contributor-covenant.org/zh-cn/version/2/1/code_of_conduct/)，版本 2.1，发布于 [https://www.contributor-covenant.org/version/2/1/code_of_conduct.html](https://www.contributor-covenant.org/version/2/1/code_of_conduct.html)。Contributor Covenant 由 [Coraline Ada Ehmke](https://coralineada.ehmke.com) 创作，遵循 [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/) 协议发布。

## 我们的标准

有助于创造积极环境的行为包括：

- 🤝 **友善与尊重**：对新人、提问者、不同背景的贡献者使用欢迎语气
- 🎯 **就事论事**：批评针对行为/代码/想法，而非个人
- 🌍 **多语言包容**：本项目以中文为主，欢迎英文；遇到不熟悉的外语，请耐心或用翻译工具
- 📚 **愿意教导**：分享知识、解释"为什么"，而非只说"这样不行"
- 🧘 **承担错误**：错了就道歉、修正、学习，不甩锅
- 🛡️ **为社区守门**：看到不当行为时主动指出，或报告给维护者

不被接受的行为包括：

- ❌ 任何形式的性化语言或图像、不受欢迎的性关注
- ❌ 挑衅、侮辱、贬损评论、人身或政治攻击
- ❌ 公开或私下骚扰（包括持续性的不当关注、跟踪、恐吓）
- ❌ 未经明确许可发布他人私人信息（如住址、电子邮箱）
- ❌ 在专业场合中可以被合理视为不当的其他行为

## 维护者责任

项目维护者（[moreg](https://github.com/moreg) 及受其委托的协作者）有责任：

- 明确社区标准，并对违规行为做出**及时且公正**的回应
- 公正地审核 Pull Request、Issue、Discussion
- 编辑、删除、拒绝违反本准则的评论、提交、代码、Issue、PR 等
- 必要时将贡献者**临时或永久禁言**

维护者在删除、警告、禁言等决定上拥有最终裁量权；但承诺每项决定都会**给出理由**（公开场合下，如合规）。

## 适用范围

本准则适用于：

- 项目仓库下的所有空间（Issues、PRs、Discussions、Wiki、Code 评论、Commit Messages）
- 项目的官方社交媒体账号、聊天群（QQ/微信/Discord）
- 当个人代表项目出席线下/线上活动时

代表项目包括：使用官方项目邮箱、通过官方账号发社交媒体、作为项目代表在活动中发言等。

## 举报违规

⚠️ 任何**安全相关**的违规（详见 [SECURITY.md](https://github.com/moreg/codeArchaeology/security/advisories/new) 走 GitHub Security Advisories 渠道）。

非安全类的行为问题，通过以下方式私下举报：

- 邮件：[在 GitHub 个人主页获取](https://github.com/moreg)（避免在仓库公开 Issue 讨论具体事件）
- GitHub 直接 @ [moreg](https://github.com/moreg)

举报应包含：

- 事件发生的时间、地点（哪个 Issue / PR / Discussion / 群）
- 涉及的人员（用户名 / 昵称）
- 事件描述（尽量客观、可验证）
- 你的联系方式（仅维护者可见）

**所有举报都会被认真对待**。维护者承诺：

- 24 小时内首次响应
- 严格保密举报人身份
- 调查结束前不公开事件细节
- 给出处理结果（无论是否处罚违规者）

## 处理流程概览

```
1. 举报     →  2. 维护者评估     →  3. 私下沟通     →  4. 处置
   (你)        (24h 内首次回应)     (与相关方)        (警告/编辑/禁言/移除)
                                                     ↓
                                                5. 公示（如必要）
                                                   公开声明、不暴露举报人
```

## 常见问题

**Q：代码评审中我不同意某个设计，能说"这个设计很烂"吗？**  
A：可以批评**设计**，但避免对**人**做价值判断。✅"这个 API 设计违反了 REST 语义" ❌"你这个设计很烂"。

**Q：维护者不同意我的 PR，能要求说明吗？**  
A：当然。任何 merge、close、修改决定，维护者都会给出理由。如不认同可礼貌追问，或发 Discussion 公开讨论。

**Q：被举报了会怎样？**  
A：维护者会先私下与你沟通，听取你的版本再决定。如确实违规，处置会考虑情节轻重（首次警告 → 临时禁言 → 永久移除）。

**Q：群内有人刷屏/广告呢？**  
A：群管理员可警告 → 移出。本准则不覆盖纯群务，但精神一致。
